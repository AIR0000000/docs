## Customer

Customer object holds the customer mobile number and the customer bill parameters.

| parameter                | type                               | description                                                                                        | example                                       |
|--------------------------|------------------------------------|----------------------------------------------------------------------------------------------------|-----------------------------------------------|
| `mobile`                 | `string` (6, 10 and 20 digits)     | Customer Mobile Number                                                                             | 9876543210, 357220                            |
| `billParameters`         | `{ name: string, value: string}[]` | Bill Parameters. The parameters to send is found in the biller details API.                        | [{"name": "Loan Number", "value": "1895159"}] |
| `billParameters[].name`  | `string`                           | Name of the bill parameter. name should match the parameter in biller details API.                 | "Loan Number"                                 |
| `billParameters[].value` | `string`                           | Value of the bill parameter. Value is validate according to the regex found in the biller details. | "1895159"                                     |

## Failure Reason

| parameter | type                           | description                                                                                        | example                                                           |
|-----------|--------------------------------|----------------------------------------------------------------------------------------------------|-------------------------------------------------------------------|
| code      | `string`                       | Error codes includes Application, Banking and BBPS error codes                                     | `"invalid-amount"`                                                |
| message   | `string`                       | Error code description                                                                             | `"Amount is invalid for this payment. Exactness does not match."` |
| type      | `FUND_TRANSFER \| BBPS \| APP` | Error Type. BBPS errors are due to BBPS, and code and message field contain the actual BBPS error. | `"APP"`                                                           |
